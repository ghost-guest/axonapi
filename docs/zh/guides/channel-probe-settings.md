# 渠道独立探测设置说明

本文说明 AxonHub 渠道级探测设置的使用方式、调度规则，以及本次列表页兼容性修复的背景。

## 功能概述

本次调整后，探测配置从“系统统一控制”扩展为“每个渠道可单独设置”：

- 每个渠道可以单独开启或关闭探测
- 每个渠道可以单独设置探测间隔
- 支持 `fixed` 固定间隔
- 支持 `random` 随机间隔
- 未单独配置时，仍可回退到原有系统默认和历史 `probeFrequency` 行为

## 界面入口

在管理界面中：

1. 进入 **Channels（渠道）**
2. 找到目标渠道
3. 打开该渠道的操作菜单
4. 点击 **探测设置**

也可以在渠道编辑弹窗中直接配置探测相关字段。

## 配置项说明

### 1. 是否启用探测

- 开启后，该渠道会参与自动探测调度
- 关闭后，即使系统级探测已启用，该渠道也会被跳过

### 2. 探测模式

支持以下三种状态：

- 跟随系统：不单独指定间隔，继续使用系统默认或历史兼容逻辑
- `fixed`：使用固定间隔
- `random`：使用随机区间

## 固定间隔模式

当探测模式为 `fixed` 时，需要设置一个固定间隔。

例如：

- 每隔 1 小时探测一次
- 每隔 1 天探测一次
- 每隔 30 分钟探测一次

要求：

- 固定间隔必须大于 0

## 随机间隔模式

当探测模式为 `random` 时，需要设置最小间隔和最大间隔。

例如：

- 最小 30 分钟，最大 2 小时
- 最小 1 小时，最大 1 天

系统行为：

- 每次进入下一轮探测调度时
- 系统会在用户指定的最小值和最大值之间随机取一个间隔
- 每次取值都可能不同，不是固定值

要求：

- 最小间隔必须大于 0
- 最大间隔必须大于 0
- 最大间隔必须大于或等于最小间隔

## 兼容行为

为了兼容历史数据，当前实现保留了以下行为：

- 如果渠道没有设置独立探测模式，则会回退到系统默认配置
- 如果历史数据仍使用旧的 `probeFrequency`，系统会继续兼容
- 历史未设置的秒级字段不会再影响渠道列表加载

## 列表页报错修复

之前列表页出现过类似以下错误：

- `probeFixedIntervalSeconds: Invalid input`
- `probeRandomMinIntervalSeconds: Invalid input`
- `probeRandomMaxIntervalSeconds: Invalid input`

根因是：

- 后端对“未设置”的秒数字段返回了 `0`
- 前端把这些字段按“正整数”校验
- 导致渠道列表查询结果在前端解析阶段失败

本次修复做了两层处理：

1. 后端在未设置时返回 `null`
2. 前端对历史 `0` 值做兼容处理，按“未设置”解析

这样可以同时兼容旧数据和新配置。

## 适用建议

- 需要稳定节奏时，使用 `fixed`
- 需要降低周期规律性时，使用 `random`
- 对暂时不想参与探测的渠道，直接关闭探测即可

## 注意事项

- 随机区间建议保留合理跨度，避免最小值和最大值过于接近
- 若渠道数量较多，建议结合系统默认配置统一规划基线策略
- 若历史数据来自旧版本，首次保存后将逐步转换为新的渠道级配置方式
